# ALT Linux Domain Debug Toolkit

Набор скриптов для диагностики проблем доменной авторизации на клиентских машинах под ALT Linux 10.4 К / 11 К.

Основная цель — получить максимально подробную картину того, что происходит:
- при старте системы;
- во время входа пользователя под доменной учетной записью;
- при применении групповых политик (GPO);
- при работе SSSD / Winbind / Kerberos;
- при сетевых обращениях к контроллеру домена;
- при блокировках со стороны антивируса, firewall или PAM.

---

# Для чего нужен этот набор

В Linux-доменах проблемы авторизации часто выглядят одинаково:

- долгий вход в систему;
- черный экран после логина;
- зависание LightDM;
- временная недоступность домена;
- intermittent login;
- "user not known";
- timeout Kerberos;
- невозможность применения GPO;
- вход локальным пользователем работает, доменным — нет.

Обычные системные логи редко дают полную картину.

Поэтому данный набор:
- собирает сетевой трафик;
- сохраняет журналы SSSD/Samba;
- отслеживает проблемные сервисы;
- фиксирует подключения;
- помогает понять, какой процесс мешает входу;
- позволяет сравнивать разные сценарии тестирования.

# Состав набора

| Скрипт | Назначение |
|---|---|
| alt-domain-debug.sh | Полная диагностика состояния системы и домена |
| alt-login-trace.sh | Трассировка процесса входа пользователя |
| find-blockers.sh | Поиск ошибок и блокирующих компонентов |
| network-watch.sh | Мониторинг сетевых соединений |

# Полезные команды
---
## Проверка домена

``` bash
realm list
```
---
## Проверка пользователя
``` bash
id DOMAIN\\\user
```
---
## Проверка GPO
``` bash
sssctl user-checks DOMAIN/user
```
---
## Проверка Kerberos
``` bash
kinit DOMAIN/user
klist
```
---
## Проверка DNS
``` bash
dig _ldap._tcp.dc._msdcs.DOMAIN SRV
```

# Практический совет
## Самая полезная диагностика получается при сравнении сценариев:

| Сценарий                          | Что проверяем      |
| --------------------------------- | ------------------ |
| без GPO / с GPO                   | влияние политик    |
| без Kaspersky / с Kaspersky       | влияние антивируса |
| локальный / доменный пользователь | проблемы PAM/SSSD  |
| успешный / неуспешный login       | различия трафика   |

---

## Данный набор предназначен не только для сбора логов, а для полноценной трассировки процесса доменной авторизации.

Главная цель:
 - увидеть весь путь логина;
 - понять, куда обращается клиент;
 - определить, на каком этапе происходит сбой;
 - выявить процесс или службу, мешающую входу в систему.
