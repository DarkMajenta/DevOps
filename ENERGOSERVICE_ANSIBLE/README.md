# ENIP2 Infrastructure Automation Research

Исследовательский проект по автоматизации управления оборудованием компании ЭНИП-2.

## Цель проекта

Проверить возможности массового управления оборудованием ЭНИП-2 с использованием современных инструментов автоматизации инфраструктуры:

* Ansible
* Python
* SNMP
* Infrastructure as Code
* Configuration Management

## Проблема

На момент начала исследования в открытой документации отсутствует информация о наличии:

* официальной Ansible Collection;
* REST API для управления устройствами;
* средств централизованного массового конфигурирования;
* интеграции с современными DevOps-инструментами.

Большинство операций предполагается выполнять через штатное программное обеспечение производителя и локальные механизмы конфигурирования.

Для эксплуатации крупных инсталляций такой подход существенно усложняет сопровождение и масштабирование инфраструктуры.

## Задачи исследования

* анализ доступной документации;
* исследование сетевых протоколов управления;
* анализ взаимодействия штатного программного обеспечения с устройствами;
* выявление возможностей автоматизации конфигурации;
* поиск существующих интерфейсов управления (SNMP, TCP, HTTP, Modbus, IEC-104 и др.);
* разработка сценариев массового управления оборудованием.

## План работ

### Этап 1

Сбор и анализ документации.

* руководство пользователя;
* технические описания;
* сетевые спецификации;
* MIB-файлы;
* сведения о поддерживаемых протоколах.

### Этап 2

Исследование взаимодействия штатного ПО с устройствами.

* анализ сетевого трафика;
* захват пакетов;
* построение карты протоколов.

### Этап 3

Reverse Engineering клиентского программного обеспечения.

* анализ исполняемых файлов;
* исследование форматов конфигураций;
* исследование механизмов обмена данными.

### Этап 4

Разработка средств автоматизации.

Возможные варианты:

* Python SDK;
* Ansible-модули;
* Ansible Collection;
* Terraform Provider (если применимо).

## Возможные результаты

* автоматизированное массовое изменение конфигурации;
* централизованное резервное копирование конфигураций;
* автоматическая инвентаризация устройств;
* интеграция с системами мониторинга;
* построение CI/CD-подходов для инфраструктуры промышленной автоматизации.

## Статус

Исследование находится на стадии первичного анализа архитектуры и интерфейсов управления.
